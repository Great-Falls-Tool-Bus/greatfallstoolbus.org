# CLAUDE.md: greatfallstoolbus.org (Claude Code overlay)

This is a thin, Claude Code-specific overlay for a sister site spawned from
`tinyland-inc/site.scaffold`. `AGENTS.md` is the canonical, provider-neutral
operating contract; read it first for every operational detail (Just
recipes, Bazel/Flywheel wiring, gitleaks, Skeleton pin, agent routes, and
the parent scaffold link). This file does not duplicate that contract.

Claude Code-specific note:

- Project skills resolve via `.claude/skills/*`, a symlink mirror of the
  canonical `.agents/skills/*`. Edit the canonical copy, not the mirror.

## Consolidation status (TIN-2533)

The agent contract is already consolidated onto `.agents/` + `AGENTS.md` as the
single source of truth. The remaining Claude-specific surfaces are
non-duplicative compatibility shims — not duplicated content — so do **not**
delete them:

- `.claude/skills/*` — symlinks into `.agents/skills/*` (zero duplicated bytes).
  The `gftb-mail-laceup-*` links are generated by `scripts/build-agent-skills.mjs`
  and `just skills-check` fails on any diff; the `tinyland-*` links are Claude
  Code's discovery path. Removing either breaks the `skills-check` CI gate.
- `CLAUDE.md` — this file. `scaffold-doctor` audits `AGENTS.md`/`CLAUDE.md`
  presence; keep it a thin pointer rather than deleting it.
- `.claude-plugin/marketplace.json` + `plugins/scaffold-core/*` — the
  git-subdir-installable `scaffold-core` plugin, documented in `AGENTS.md` and
  linked from the public `/agent` route. Whether a spoke re-ships it is an
  operator decision, not a mechanical cleanup.
